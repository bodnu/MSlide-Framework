# Contributing

Thanks for helping improve MSlide³ Framework. This repository is the public starter package, so contributions should keep the starter clear, stable, minimal, and easy to preview locally.

## Issue Reporting

When reporting an issue, include:

- What you expected to happen
- What actually happened
- Browser and operating system
- Whether you served the project through localhost
- Any relevant console or network errors

`file://` is not a supported preview mode. Please verify issues through a local server before reporting them.

## Pull Requests

Pull requests should be focused and easy to review.

- Keep changes scoped to the issue or improvement.
- Avoid unrelated cleanup.
- Explain any path, asset, or documentation changes.
- Verify the starter through localhost when a change could affect preview behavior.

## Documentation Improvements

Documentation improvements are welcome, especially when they clarify:

- Quick start steps
- Folder purpose
- Starter boundaries
- Release packaging
- Runtime preservation expectations

## Runtime Preservation Principles

- Preserve the blank starter runtime.
- Preserve vertical navigation and sequential scroll structure.
- Do not add tutorials, showcases, products, or client examples to the starter.
- Do not replace dependencies without a dedicated dependency review.
- Do not restructure folders without a dedicated migration plan.
- Keep `MSlide2-Framework/` untouched when working on MSlide³.
- Keep `mslide2/4/` untouched when working on this public starter.

## Archive Before Delete

Do not delete project files permanently during cleanup work. Move excluded material into `_archive/` first, preserving original paths wherever possible.
